class PilhaInt{
    public:
        PilhaInt(); // Construtor
        PilhaInt( const PilhaInt& p ){ // Construtor de cópia: sempre fazer const classe ref
        cout << "Chamou construtor de cópia" << endl;
            atual = p.atual; // o primeiro atual é o do novo objeto criado
            for (int i = 0; i < p.atual; i++){
                tab[i] = p.tab[i];
            }
        }

        const PilhaInt& operator= ( const PilhaInt& p) { // sobrecarga de operadores! deve ser sempre const porque não faz sentido mudar atribuição; não pode mudar numero de operadores na atribuição, por isso um parametro
            if ( this != &p ){ // se é o mesmo objeto, não precisa atribuir -> além de ineficiente, se você usa ponteiro dá ruim --> sempre verificar se this != &p
                atual = p.atual; // o primeiro atual é o do novo objeto criado
                for (int i = 0; i < p.atual; i++){
                    tab[i] = p.tab[i];
                }
            }
            return p;
        }

        void imprime( ostream& o ) const {
            for (int i = 0; i < atual; i++){
                o << p.tab[i] << " ";
            }
        }
}

int main(){
    PilhaInt um, outro;
    outro = um; // faz tab[i] = p.tab[i]; por nós -> operador de atribuição

Templates
-> Programação genérica: escrever código que funciona corretamente para tipos arbitrários
-> É um poliforfismo estático, que ocorre durante compilação (não na execução)
-> Padrão (molde) para criar funções -> no assembly, cada molde aplicado a um tipo cria uma função para cada combinação de tipos em parâmetros

ex: template <typename T> // class <=> typename
    inline const T& Max (const T& a, const T& b){ // inline é para não chamar função, e sim colocar direto no código -> se fiver um for não vai fazer inline
        // const ref para não fazer uma cópia desnecessária, se os objetos são pequenos, não precisa
        return a > b ? a : b;
    }

    Max(3,5);         // int
    Max(3.14, 2.71);  // double
    Max("abc", "xyz"); // string -> string não tem >, então o que será feito é comparação dos endereços onde essas strings estão
    Max(1, 1.1);       // vai dar erro, para corrigir precisamos precisamos passar o tipo específico, como em Max<int>(6.8, 6.9), que aí converte para int o resultado


OBS1: Se você for fazer sobrecarga de função já definida em std, como "max", vai dar erro. Para tratar isso, podemos ou marcar como escopo global ::max(1,2), ou definir nosso namespace
